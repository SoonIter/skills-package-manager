# Introduction

skills-package-manager is a package manager for agent skills. It manages, installs, updates, and links skill directories based on `SKILL.md`.

It currently provides two core parts:

- `skills-package-manager`: the core library and the `spm` CLI.
- `pnpm-plugin-skills`: a pnpm plugin that automatically synchronizes skills during `pnpm install`.

## Why it exists

As AI coding agents start to rely on more and more skills, teams usually run into several problems:

- Skill sources are scattered across GitHub, local directories, or internal team repositories.
- There is no unified entry point for skill installation, so local environments can easily drift.
- Skills need to be linked into directories such as `.agents/skills` and `.cursor/skills`, and manual synchronization is error-prone.
- Without a lockfile, team members and CI may end up with inconsistent context.

skills-package-manager solves these problems with a familiar “manifest + lockfile” workflow.

## Core capabilities

- **Unified declaration**: Declare the skills to install and their target directories in `skills.json`.
- **Deterministic installation**: Lock git commits or local content digests in `skills-lock.yaml`.
- **Multiple source types**: Support GitHub shorthand, Git URLs, specifiers with path/ref, and local `file:` skills.
- **Automatic linking**: Sync installed skills into one or more agent skill directories automatically.
- **Incremental updates**: `spm update` refreshes only git-based skills without rewriting the manifest.
