**skills-package-manager** is a specialized tool designed to manage [SKILL.md](https://github.com/SoonIter/skills-package-manager/blob/main/AGENTS.md)-based agent skills. It brings the power and reliability of modern package management (like `npm` or `pnpm`) to the world of AI agents.

## Core Philosophy

Traditional agent skills often involve manual directory management and hard-to-track versions. **SPM** changes that by focusing on:

- **Version Control**: Use a lockfile (`skills-lock.yaml`) to ensure every developer on your team and every CI runner uses the exact same skill versions.
- **Protocol Flexibility**: Install skills from anywhere—Git repositories, NPM packages, local symlinks, or even `.tgz` archives.
- **Zero Duplication**: Install a skill once and automatically link it to multiple agent environments (Claude, Cursor, custom agents) simultaneously.
- **Developer First**: Fully compatible with existing tools like `npx skills` and deeply integrated with `pnpm`.

## Project Structure

The ecosystem consists of two primary components:

- **`skills-package-manager`**: The core engine and the `spm` CLI tool.
- **`pnpm-plugin-skills`**: A first-class pnpm integration that hooks into the dependency resolution lifecycle.
